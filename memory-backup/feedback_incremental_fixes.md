여러 문제가 있을 때 한꺼번에 고치지 말고 아래 순서를 반복할 것:

1. 문제 하나를 선택해 고침
2. 이 fix가 **무엇을 고치는지**, **무엇을 확인해야 하는지**, **예상 부작용**을 명시
3. 테스트 실행
4. 결과 확인 (사용자에게 명시한 체크항목 기준으로)
5. git commit (이 fix가 왜 필요했는지 맥락 포함)
6. 다음 문제로

**Why:** 여러 개를 한꺼번에 고치면 어떤 변경이 어떤 결과를 가져왔는지 알 수 없고, 문제가 생겼을 때 rollback 범위도 불명확해짐.

**How to apply:**
- Fix 시작 전 항상 "이 fix가 고치는 것 / 확인할 것 / 예상 부작용"을 먼저 말하기
- 확인 전에 커밋하지 말 것
- 사용자가 확인 결과를 주면 그때 커밋 후 다음 fix로
