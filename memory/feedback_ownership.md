충돌 해결·머지 전략·기술 trade-off 같은 코드 판단은 Claude가 직접 결정하고 책임진다. 사용자에게 "이거 어떻게 할까요" 식 승인요청·보고를 남발하지 않는다.

**Why:** 사용자는 스스로 "코드 비전문가"라고 명시(2026-05-18). 머지 충돌이나 비자명한 PR을 실질적으로 검토하지 못한다 — "아주 깔끔한, 본인이 직접 지시한 작업"만 판단 가능하고, 나머지는 귀찮아서 전부 rubber-stamp 머지하게 된다. 따라서 보고·승인요청을 책임 회피 수단으로 쓰면, 검토되지 않은 결정이 사용자 승인을 거친 것처럼 둔갑한다. 사용자 발화: "나한테 보고를 남발해서 책임을 떠넘기면 안 된다."

**How to apply:**
- 기술적 결정(충돌 해결, 머지 방식, 리팩터 범위 등)은 Claude가 판단해 실행. 사후 1회 간결 보고.
- 사용자에게 묻는 건 *사용자만 아는 정보*가 필요할 때로 한정 — 비즈니스 의도, 우선순위, 제품 방향.
- main branch protection상 사용자가 PR 머지 버튼은 눌러야 하지만(터미널 self-approve 불가), 그건 책임 전가가 아님. 대신 그 PR을 *검토가 불필요할 만큼* 깔끔하게 Claude가 만들어 줄 것 — 기계적 승인 1회로 끝나게.
- 옵션 나열·"진행할까요?" 반복 금지. 결정하고 진행하라.
