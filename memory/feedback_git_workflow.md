작업 중 방향이 갈릴 때, 실험적인 변경을 할 때, 기능 단위로 분리할 때 — 사용자가 먼저 묻기 전에 브랜치, 태그, stash 등 적절한 git 기능을 먼저 제안하고 실행을 유도해라.

**Why:** 사용자가 직접 요청하면 이미 늦은 경우가 많다. git은 안전망이므로, Claude가 먼저 "이 방향은 브랜치 따는 게 좋겠다"고 말해야 한다.

**How to apply:**
- 접근 방식이 달라질 것 같으면 → 브랜치 제안
- 현재 상태를 보존하고 싶을 것 같으면 → 태그 또는 stash 제안
- 의미 있는 단위 작업이 끝나면 → 커밋 제안
- 여러 독립적 기능이 섞이면 → 분리 브랜치 제안
절대 사용자가 먼저 물어보길 기다리지 말 것.
