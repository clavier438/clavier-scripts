사용자가 작업을 진행하라고 하면 "빠른 A vs 정석 B vs 시뮬레이션 C" 같은 옵션 메뉴 제시 금지. 그냥 클린아키텍처/SOLID/단계적 원칙대로 끝까지 처리.

**Why:** 2026-05-14 사용자 폭발: "빠른게 문제가아니고 똑바로 처리하라니까요? 뭔자꾸 빨리빨리거려요 그냥 똑바로 처리하라고하는데". 결정해야 할 trade-off 없는 단순 작업에서 굳이 사용자에 선택권 떠넘기는 패턴.

**How to apply:**
- "전체 진행", "다 처리", "순리대로", "스스로 판단" 같은 발화 = 옵션 제시 단계 끝. 그냥 수행.
- 옵션 제시는 *진짜 트레이드오프* 있는 결정에만 (예: API 디자인 두 갈래 / 데이터 마이그레이션 전략 / 의미 손실 가능한 destructive 작업).
- 검증/테스트도 옵션이 아님 — 클린아키텍처 원칙상 *해야 하는 것*이면 그냥 함.
- "추천: B → A" 같은 의사결정 권유로 위장한 옵션 제시도 금지.
