도구·CLI·브리핑을 만들거나 개선할 때, 현재 상태(진행 중/완료/대기/에러 등)를 별도 명령이나 메뉴 선택 뒤에 숨기지 말고 **진입하자마자 기본 출력에 표시**할 것.

**Why:** 사용자 발화 (2026-05-16): "스테이터스를 물어보면 표기하는게 아니라 그냥 바로 브리핑부터 얘기해줬으면좋겠어 — 지금 상태를 알아야, 뭔가를 할 수 있으니까." 상태를 알아야 다음 행동을 결정할 수 있는데, 상태가 별도 조회 뒤에 있으면 사용자가 매번 한 단계를 더 거쳐야 함.

**How to apply:** 새 도구/명령/브리핑 화면 설계 시 — 현재 상태를 first-class로 진입 화면에 노출. 예: workerCtl 무인자 브리핑에 워커별 push 상태 한 줄 추가. 상세 패널은 별도 명령으로 두되, 요약 상태는 항상 먼저 보이게.
