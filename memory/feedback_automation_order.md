자동화는 과정의 마지막 단계다. 절대 첫 단계가 되어서는 안 된다.

**Why:** 이 실수의 교훈 — formula/lookup/rollup 필드 자동 생성 스크립트를 세션 내내 짰는데, 근본 설계(두 사슬의 빡빡한 결합)가 틀려서 전부 폐기됨. 패턴을 모르는 상태에서 자동화하면 잘못된 것을 자동화하게 됨.

**How to apply:**
- Claude는 유연한 1단계 실행자 — 패턴 불명확한 초기엔 Claude가 직접 처리
- 사용자가 "스크립트로 만들자"고 하면 → "Claude가 이미 직접 해봤는가? 패턴이 잡혔는가?" 먼저 확인
- 패턴 검증 안 된 걸 바로 스크립트화하려 하면 → 경고: "먼저 제가 직접 처리해보면서 패턴을 잡는 게 낫겠습니다"
- 두 시스템을 연결하는 코드 짜기 전 → 결합 방식 먼저 검토: "한쪽이 바뀌면 다른 쪽도 깨지는 구조인가?"

올바른 순서 (Elon Musk 알고리즘):
1. 요구사항을 의심한다
2. 불필요한 단계를 제거한다
3. Claude가 직접 처리 — 패턴이 보일 때까지
4. 속도를 높인다
5. 자동화 — 마지막으로, 안정된 패턴만
