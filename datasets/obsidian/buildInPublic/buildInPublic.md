## 0. 이 문서의 목적

빌드 인 퍼블릭을 "왜 하는지", "어떻게 할 건지", "실제로 뭘 했는지"를 **내 머릿속 사고 과정 그대로** 기록하는 문서.

트위터에 올리는 건 정제된 1줄이지만, 여기에는 날것의 생각·판단 근거·실패 원인 분석을 전부 남긴다.

**규칙:**

- 매일 오전 루틴 끝나고 5분 안에 여기에 한 줄이라도 적기
- 예쁘게 쓰려고 하지 말 것. 생각 흐름 그대로.
- 숫자는 반드시 포함 (리드 수, 회신율, 매출 등)

## 1. 왜 빌드 인 퍼블릭을 하기로 했는가

**내 상황 정리:**

- 고객 0곳, 매출 0원인 완전 초기 단계
- 혼자 하는 사업이라 외부 압박이 없으면 쉽게 미루게 됨
- "오전 4시간만 일하고 월 500" 이라는 목표는 매력적인 스토리가 될 수 있음

**빌드 인 퍼블릭을 선택한 진짜 이유:**

1. **자기 압박 장치** — 공개했으니 "이번 주 아무것도 안 했다"를 쓰고 싶지 않은 마음이 생김. 이게 최고의 동력.
2. **마케팅 비용 0원** — 광고비 없이 콘텐츠 자체가 리드를 만들어줄 가능성
3. **신뢰 구축** — 과정을 투명하게 보여주면 "이 사람은 진짜 하는 사람"이라는 인식이 생김
4. **칸반과 찰떡** — 칸반 보드에서 카드가 이동하는 것 자체가 콘텐츠가 됨. 별도로 "콘텐츠를 만드는" 시간이 거의 안 듦.

**우려 사항:**

- 콘텐츠 제작 자체가 병목이 되면 안 됨 → **매일 10분, 주 1회 30분** 제한 엄수
- 호텔 고객 정보 노출 금지 → 숫자만 공개, 이름·연락처·대화 내용 절대 비공개

## 2. 채널 전략 — 내 판단 근거

**결정: X(트위터) 메인 + 노션/브런치 보조**

왜 X가 메인인가:

- 빌드 인 퍼블릭 커뮤니티가 가장 활발한 곳
- 짧은 글 + 숫자 공개 포맷이 오전 4시간 근무와 맞음 (길게 쓸 시간 없음)
- 해시태그(#빌드인퍼블릭, #BuildInPublic)로 같은 길 가는 사람들과 자연스럽게 연결

왜 노션/브런치 보조인가:

- 주간 리포트처럼 긴 글은 X에 안 맞음
- 나중에 "Week 1\~52 전체 여정" 같은 아카이브로 활용 가능
- SEO 효과로 검색 유입도 기대

왜 LinkedIn·YouTube·인스타는 나중인가:

- LinkedIn: B2B 감성이라 호텔 업계 관계자가 볼 수 있지만, 초기에 채널 2개 이상은 부담
- YouTube/릴스: 임팩트 크지만 영상 편집 시간이 칸반 WIP를 터뜨림
- 인스타: 호텔 비주얼과 맞지만 빌드인퍼블릭 포맷과는 궁합이 안 맞음

**판단 기준:** "10분 안에 올릴 수 있는가?" → X만 통과

## 3. 콘텐츠 설계 — 내 사고 과정

### 3-1. 매일 트윗 포맷을 왜 이렇게 정했나

`[Day N] 한 줄 요약 + 숫자 1개`

이 포맷의 핵심은 \*\*"숫자 1개"\*\*에 있다.

- 숫자가 있으면 사람들이 멈춰서 읽음
- 숫자가 있으면 나중에 추이를 비교할 수 있음
- 숫자가 있으면 나 자신도 "오늘 진짜 뭘 했는지" 속일 수 없음

예시 흐름:

- Day 1: 리드 36곳 등록
- Day 7: 5곳 컨택, 회신 1곳 (20%)
- Day 14: 첫 미팅 완료, 견적 150만 제안
- Day 30: 월 매출 0 → 150만

### 3-2. 주간 리포트 구조를 왜 이렇게 정했나

1. 이번 주 한 줄
2. 숫자 (리드/컨택/미팅/계약/매출)
3. 칸반 스냅샷
4. 병목
5. 배운 것
6. 다음 주 목표

**핵심 판단:** 3번 "칸반 스냅샷"이 가장 중요.

- 보드 캡처 한 장이면 "어디서 막혔는지"가 한눈에 보임
- 나도 매주 병목 패턴을 추적할 수 있음
- 독자도 "아 이 사람 이번 주 상담 단계에서 막혔구나" 바로 이해

### 3-3. 공개/비공개 기준 — 왜 이렇게 나눴나

**공개 O:** 리드 수, 회신율, 매출, 소요 시간, 도구, 워크플로우, 칸반 구조\
**공개 X:** 호텔명, 담당자, 연락처, 대화 내용, 계약 조건

판단 근거:

- "과정"은 공개해도 경쟁자가 베낄 수 없음 (내 실행력이 핵심이지 정보가 아님)
- "고객 정보"는 공개하면 신뢰를 잃음 (호텔 사장님 입장에서 자기 정보가 돌아다니면 불쾌)
- 숫자를 공개하면 나중에 "월 500 달성" 스토리가 더 신빙성 있음

## 4. 칸반 → 콘텐츠 자동 생성 원리

이게 이 전략의 **가장 핵심적인 아이디어**.

칸반 보드에서 카드가 움직일 때마다 → 그게 곧 트윗 소재가 됨.

| 보드 이벤트 | 트윗 변환 |
| --- | --- |
| 카드 → 컨택 발송 | "\[Day N\] N곳에 제안 메일 발송. 이번 주 회신율 X%" |
| 카드 → 상담 중 | "\[Day N\] 첫 미팅 성사! 문제: OO, 제안: XX" |
| 카드 → 계약 진행 | "\[Day N\] 계약 N호 진행 중. 누적 MRR +Xk" |
| WIP 초과 발생 | "\[Day N\] 병목 발견. 상담 2건 동시에 멈춤. 원인: OO" |
| 카드 → 런칭 완료 | "\[Day N\] N번째 호텔 사이트 런칭! (스크린샷)" |
| 카드 → 운영 & 업셀 | "\[Day N\] 업셀 제안 OO. 반응: OO" |

**왜 이게 좋은가:**

- 콘텐츠를 "따로 만드는" 시간이 거의 0에 수렴
- 칸반을 제대로 운영하면 콘텐츠가 자동으로 생산됨
- 즉, 일을 하는 것 = 콘텐츠를 만드는 것. 별도 작업이 아님.

>

> 같은 템플릿, 같은 구조를 여러 호텔에 복제 적용하는 모델.
>
> 현재 상태:
>
> - HubSpot에 리드 36곳 등록
> - Jira 칸반 보드 세팅 완료
> - 첫 컨택 준비 중
>
> 매일 진행 상황을 여기에 공개합니다.\
> 과정, 숫자, 실패 전부 투명하게.
>
> #빌드인퍼블릭 #1인사업 #부티크호텔 #노코드

**이 포스트의 설계 의도:**

1. "오전 4시간, 월 500" → 후크. 사람들이 궁금해하는 조합
2. "부티크 호텔 전용 웹사이트" → 니치가 명확해서 전문성이 느껴짐
3. "같은 템플릿을 복제 적용" → 비즈니스 모델이 한 줄로 이해됨
4. "현재 상태" 숫자 → "이 사람 진짜 시작했구나" 증거
5. "실패 전부 투명하게" → 빌드인퍼블릭의 핵심 약속
