# # 4월 전투 계획 — 500만 입금까지 거꾸로 오기

## 1. 최상위 약속 (불변)

> **"하루 4시간만 투자해서 월 500만 원을 번다."**

레퍼런스 클래스 검증 결과: 허황되지 않음.

- 솔로 웹 프리랜서: 건당 300\~500만 표준. 템플릿 모델이면 프리랜서보다 유리.
- Pieter Levels (@levelsio): 혼자 4\~6시간/일, 같은 구조 반복 적용, 월 수천만\~수억. 월 500은 보수적 목표.
- 부티크 호텔 전문 에이전시: 시각적 스토리텔링 + 예약 UX 리디자인 후 직접예약 38% 상승 사례. 시장이 존재하고 효과가 검증된 모델.
- 핸드오프 모델 기준: 월 1건 계약 × 500만 = 달성. 템플릿 숙련 후 제작 1\~2주 → 4시간/일로 충분.

## 4. 핵심 설계 원칙: 데모는 템플릿의 변주에 불과해야 한다

매번 처음부터 만들면 = 프리랜서. 시스템이 아님. 4시간 모델 성립 불가.

```
잘못된 순서: 호텔A 데모 → 호텔B 데모 → 호텔C 데모 (매번 처음부터)
맞는 순서:   템플릿(한 곡) → 호텔A 변주 → 호텔B 변주 → 호텔C 변주 (스왑만)
```

- 템플릿 없이 데모 1개 = 2\~3일
- 템플릿 있으면 변주 1개 = 반나절\~1일
- 템플릿 구축이 진짜 투자. 그 이후는 복제.

오늘 발견한 병목("에디토리얼 디자인 언어 = 한 곡인데 변주가 다채로운 느낌의 조건")이 바로 이 템플릿의 핵심 설계 문제. 이걸 풀면 변주가 빨라지고, 못 풀면 매번 처음부터 만들게 됨.

## 6. "무사하게 책임을 진다"의 조건

데모를 먼저 만들고 → 그걸로 세일즈하는 구조:

- 계약 시점에 이미 80% 만들어져 있음

"먼저 만들고 파는 것"이 리스크를 내 쪽에서 미리 소화하는 구조.

## 7. 4시간의 경계 — 장전과 발사

### 4시간 안 = 발사 (산출물)

- 특정 호텔을 위한 리서치/조사/디자인/제작
- 컨택/미팅/제안서/계약
- 빌드 인 퍼블릭 포스팅

### 4시간 밖 = 장전 (실력)

- 에디토리얼 디자인 공부, 레퍼런스 탐방
- 브랜딩/IP 관련 독서
- 타이포그래피, 여백, 흐름 감각 훈련

장전은 얼마든지 오래 해도 된다. **방아쇠 당기는 건 4시간 안에서만.** 밖에서 쌓은 것을 4시간에 쏟아내는 구조.

이 구분이 필수인 이유: "4시간 월 500" 모델을 테스트하는 중이다. 산출물을 밖에서도 하면 실험이 오염된다. 나중에 "4시간으로 됐다"고 말할 수 없게 됨.

---

## 8. 왜 4시간을 지키는가 — 바벨 전략

"4시간 월 500"은 생산성 제약이 아니라 바벨의 안전한 쪽이다.

- **4시간 = 바벨의 안전한 쪽** — 검증된 모델, 반복 가능, 월 500 바닥 확보
- **나머지 시간 = 바벨의 위험한 쪽** — IP, 새로운 실험, 큰 베팅, 자체 브랜드, 상방 무한
- **8시간 풀로 일하기 = 가운데** — 안전하지도 않고 상방도 없는 최악의 포지션

4시간이 8시간이 되면 → 위험한 쪽이 사라진다 → 상방이 닫힌다.\
바닥이 없으면 위험한 베팅 자체를 못 한다 — 생존이 위협받으니까.

## 9. 스킨 인 더 게임 — 해자와 진화

### 같은 일을 반복 치면 죽는다

템플릿 복제만 하면 → 운영 효율성 게임 → 가격 경쟁 → 해자 없음 → 대체됨.

### 해자 = "이 사람을 거치면 직접 예약이 올라간다"는 데이터

웹사이트는 누구나 만든다. "부티크 호텔 10곳에 적용해서 직접 예약 38% 상승시킨 사람" — 이건 복제 불가능.

### 일의 종류가 바뀌어야 한다

```
초기:   4시간 = 만든다 (제작)
6개월:  4시간 = 데이터 보고 판단 + 조금 만든다 (컨설팅 + 제작)
1년:    4시간 = 판단 + 온보딩 (컨설팅 + 파트너십)
2년:    4시간 = IP 설계 (전략)
```

### 브랜드 위주 비즈니스로 가야 파워가 몰린다

브랜드를 정의하고 만드는 쪽으로 능력을 키우면 → 클라이언트 거를 하다가 내 거를 하는 것도 쉬워지고, 여차하면 같이 할 수도 있다.

### 진화 경로

```
지금:    호텔 브랜드를 만들어준다 (남의 세계관)
6개월:   그 실적이 내 브랜드를 키운다 (내 세계관)
1년:     내 브랜드로 호텔이 찾아온다 (파워 역전)
2년:     호텔과 파트너/공동 브랜드 (같이 한다)
그 이후:  내 브랜드로 내 것을 한다 (독립)
```

## 10. "4시간 월 500" = 브랜드 그 자체

호텔한테 해주려는 것: "세계관을 정의해서, 그 세계관이 사람을 끌어당기게 만든다."

빌드 인 퍼블릭으로 하고 있는 것: **"4시간만 일하고 월 500 만드는 사람"이라는 세계관을 만드는 것.**

이 선언은:

- 생산성 제약이 아니라 **브랜드의 핵심 서사**
- 바벨의 안전한 쪽을 지키는 **구조적 장치**
- 빌드 인 퍼블릭의 **후크이자 콘텐츠**

이 서사가 사람을 모으고, 사람이 모이면 기회가 열리고, 기회가 열리면 일의 종류가 바뀐다.

**호텔한테 해주는 것(세계관 → 사람이 모임 → 돈이 됨)을, 자기 자신한테도 똑같이 하는 구조.**

## 11. 실패해도 된다 — 4월은 첫 번째 루프다

### 500만 못 채우면?

다음 달에 다시 하면 된다. 커뮤니티에서 패배 선언하고 잠적하는 게 아니다.

빌드 인 퍼블릭의 핵심 약속이 뭐였냐: **"과정, 숫자, 실패 전부 투명하게."**

실패하면 이렇게 쓰면 된다:

> "Day 21: 4월 500만 목표, 결과 150만. 실패. 원인은 X. 5월엔 이렇게 바꾼다."

이게 잠적이 아니라 **콘텐츠**. 사람들은 성공 스토리보다 실패 후 재구조화하는 과정에 더 끌린다.

### 프레임워크가 이걸 전제하고 있다

- 1회차: 간당간당하게 내보냄 → 실패할 수 있음
- 2회차: 진짜 피드백 기반으로 수정
- N회차: 복리로 쌓임

**1회차에서 성공하라는 프레임워크가 아니다.** 1회차는 가설 테스트. 4월은 첫 번째 루프.

레고 크레인이 첫 시도에서 떨어지는 건 실패가 아니라 **데이터 수집**. 떨어진 이유를 기록하고 구조를 바꿔서 다시 올라가면 된다.

### 진짜 파멸은 뭐냐

- 500만 못 벌었다 = **파멸 아님.** 다음 달에 다시.
- 시도 안 하고 미루다가 6개월 지남 = **이게 파멸.**
- 공개했는데 "실패했습니다" 쓰기 싫어서 잠적 = **이것도 파멸.**

4월에 150만 벌었어도 → 그 과정에서 템플릿 만들었고, 컨택 경험 쌓았고, 피드백 받았으면 → **5월이 훨씬 빨라진다.** 그게 복리.

### 한 줄
