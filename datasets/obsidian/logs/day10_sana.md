## 1. subitems.csv

**역할(추정):**  
어떤 아이템(메뉴/오퍼 등)에 딸린 **하위 옵션·세부 정보(sub item)** 리스트.

**구조:**

- _record_id: 서브아이템 고유 ID (다른 테이블에서 참조)
    
- name: 서브아이템 이름 (예: 신청 오픈, 포함 내용)
    
- items: 이 서브아이템이 연결된 상위 item들의 ID 배열(문자열 형태, 예: ["recd47iaODvMH5oA1"])
    
- notes, notes2: 설명 텍스트
    
- price: 가격 (있는 경우)
    
- caption: 짧은 캡션
    
- order: 정렬 순서

## 2. tags.csv

**역할:**  
여러 아이템/스토리에 붙는 **태그(분류) 정의 테이블**.

**구조:**

- _record_id: 태그 고유 ID
    
- name: 태그 영문 키 (예: Food, single offer)
    
- labelKr: 태그 한글 이름 (예: 음식, 단일 오퍼)
    
- db: 태그의 종류/도메인 (예: theme, offer)
    
- order: 태그 정렬 순서
    
- items: 이 태그가 붙은 item ID들의 배열(문자열)
    
- stories: 이 태그가 연결된 story ID들의 배열(문자열, 없을 수도 있음)
    

---

## 3. tag0.csv

**역할:**  
상위 카테고리/섹션 같은 **1차 분류 태그(메인 탭/블록)**.

**구조:**

- _record_id: 고유 ID
    
- name: 이름 (예: 객실, 어메니티, globals)
    
- notes: 설명
    
- slug: 슬러그/영문 코드 (예: Rooms, Amenities)
    
- caption: 짧은 설명
    
- gallery: 이 태그 섹션에 쓰일 대표 이미지들(여러 개) JSON 문자열
    
- order: 정렬 순서
    
- items: 이 섹션에 속하는 item ID들 배열(문자열)
    
- stories: 이 섹션과 연결된 story ID들 배열(문자열, 없을 수도)

## 1. 상위 섹션(tag0) 설계

시소소 사이트를 롱스크롤 원페이지 + 일부 딥링크 정도로 본다면, tag0는 대략 이렇게 잡을 수 있어:

1. Rooms (객실)
    

- name: 객실
    
- slug: Rooms
    
- 설명: 시소소의 각 객실 소개 (타입이 여러 개면 여러 item 연결)
    

2. Experiences (경험)
    

- name: 경험
    
- slug: Experiences
    
- 설명: 시소소 주변/내부에서 즐길 수 있는 경험 (바다 산책, 일출, 로컬 카페, 바비큐 등)
    

3. Dining (식사/조식)
    

- name: 다이닝
    
- slug: Dining
    
- 설명: 조식, 웰컴 드링크, 바비큐 옵션 등
    

4. Offers (오퍼/패키지)
    

- name: 오퍼
    
- slug: Offers
    
- 설명: 연박 할인, 시즌 패키지 등 단일 오퍼들
    

5. Story (시소소 스토리)
    

- name: 시소소 스토리
    
- slug: Story
    
- 설명: 왜 이곳을 만들었는지, 시소소에서의 하루, 계절 이야기 등 stories 연결
    

6. Guide (이용 안내)
    

- name: 이용 안내
    
- slug: Guide
    
- 설명: 체크인/아웃, 주차, 반려동물 가능 여부, 하우스 룰, 환불 규정 등
    

7. Globals (글로벌/공통)
    

- name: globals
    
- slug: 없음
    
- 설명: 사이트 곳곳에서 쓰이는 공통 블록(푸터, 공통 공지, 예약 안내 문구 등)

## 2. items 설계 – 어떤 아이템들을 둘 것인지

### 2-1. 메인 히어로 섹션

- items 한 레코드 예시
    
    - db: storyBlocks (또는 hero)
        
    - name: 바다와 가장 가까운 시소소
        
    - subtitle: 삼척의 파도 소리와 함께 머무는 하루
        
    - tag0: globals
        
    - subitems:
        
        - 예약하기 버튼 (CTA URL: 스테이폴리오 or 자체 예약)
            
        - 시소소 한 줄 소개
            
        - 위치/지도 요약
            

이 히어로를 페이지 최상단에 배치.

---

### 2-2. 객실 섹션 (Rooms)

각 객실 타입을 하나의 item으로:

- item 예시: 오션뷰 더블룸
    
    - db: room (혹은 menu/product 같은 형식)
        
    - tag0: Rooms
        
    - tags:
        
        - Ocean view
            
        - Couple
            
    - gallery1~N: 객실 사진들
        
    - subitems:
        
        - 기본 정보 (평형, 인원, 침대 타입)
            
        - 포함 어메니티 (타월, 어메니티 브랜드, 커피/티 등)
            
        - 요금 안내 (성수기/비성수기, 기준 인원 추가 요금 등)
            
        - 유의 사항 (금연/반려동물 등)
            

객실이 둘 이상이면, 같은 구조로 여러 item을 붙이고, order로 노출 순서를 결정.

---

### 2-3. 경험 섹션 (Experiences)

시소소 주변과 내부에서의 경험을 묶는 item들:

- item 예시1: 새벽의 동해 일출 산책
    
    - db: menu 혹은 experience
        
    - tag0: Experiences
        
    - tags: Slow travel, Nature
        
    - notes: “도보 ○분 거리 해변 산책로, 일출 시간 안내”
        
    - subitems:
        
        - 추천 시간대 (일출 기준 +-30분)
            
        - 준비물 (외투, 운동화 등)
            
- item 예시2: 삼척 로컬 카페 · 식당 지도
    
    - tag0: Experiences
        
    - tags: Local
        
    - subitems:
        
        - 추천 카페 리스트
            
        - 추천 식당 리스트
            
        - 각 장소에 대한 짧은 설명과 거리/이동 방법
            

이 섹션은 “시소소에서 머물면서 어떻게 하루를 보낼지”를 그려주는 역할.

---

### 2-4. 다이닝 섹션 (Dining)

- item 예시: 조식
    
    - db: menu
        
    - tag0: Dining
        
    - tags: Food
        
    - subitems:
        
        - 포함 내용 (시간, 구성, 제공 장소)
            
        - 추가 인원 요금
            
        - 알레르기/식단 문의 방법
            
- item 예시: 바비큐 옵션
    
    - subitems:
        
        - 포함 내용 (고기/야채 구성, 시간)
            
        - 가격
            
        - 신청 방법 (사전 연락, 마감 시간 등)
            

---

### 2-5. 오퍼/패키지 섹션 (Offers)

너가 가진 CSV의 single offer 구조와 거의 동일하게:

- item 예시: 연박 할인 패키지
    
    - db: menu
        
    - tag0: Offers
        
    - tags: single offer, Seasonal
        
    - subitems:
        
        - 포함 내용 (숙박일수, 조식/경험 포함 여부)
            
        - 신청 오픈 (기간 텍스트)
            
        - 유의 사항 (환불규정, 변경 규정 등)
            
- item 예시: 겨울 파도 시즌 오퍼
    
    - tag0: Offers
        
    - tags: Winter, Ocean view
        
    - 구조는 동일
        

이 오퍼들은 stories와도 연결해서, “스토리 읽다가 → CTA로 해당 오퍼”로 가게 만들 수 있음.

---

### 2-6. 가이드 섹션 (Guide)

- item 예시: 체크인 · 체크아웃 안내
    
    - db: globals 혹은 guide
        
    - tag0: Guide
        
    - subitems:
        
        - 체크인
            
        - 체크아웃
            
        - 셀프 체크인 여부
            
- item 예시: 이용 수칙
    
    - 소음, 흡연, 주차, 반려동물, 시설 이용 시간 같은 내용을 subitems로 분리
        
- item 예시: 교통 안내
    
    - 대중교통/자차로 오는 법, 주차 위치 등
        

이 섹션은 사이트 하단 또는 별도 “이용 안내” 페이지로도 링크.

## 3. tags 설계 – 필터/테마

tags 테이블은 다음처럼 쓰면 좋아:

- Theme 태그:
    
    - Ocean view (오션뷰 관련 객실/스토리/오퍼 전부에 태그)
        
    - Slow travel
        
    - Local (로컬 경험, 식당/카페 등)
        
- Audience 태그:
    
    - Couple
        
    - Family
        
    - (필요하면) Pet-friendly
        
- Offer 태그:
    
    - single offer (단일 오퍼)
        
    - Seasonal (계절 오퍼)
        

이렇게 해두면,

- “커플 여행” / “오션뷰 좋아하는 사람” / “겨울 시즌” 이런 기준으로 프론트에서 필터링 가능.

## 4. stories 설계 – 시소소만의 이야기

stories는 시소소 브랜딩을 강화하는 에디토리얼 콘텐츠:

1. 스토리: 시소소에서의 하루
    

- tag0: Story
    
- tags: Slow travel, Ocean view
    
- items:
    
    - 객실 item(오션뷰 룸)
        
    - 경험 item(일출 산책, 로컬 카페 지도)
        
    - 다이닝 item(조식)
        
- 사이트에서는 “체크인 → 오후 시간 → 저녁 → 밤 → 아침”처럼 하루의 흐름으로 보여주면서, 각 단계에서 관련 item들을 끌어다 씀.
    

2. 스토리: 계절마다 다른 시소소의 바다
    

- tags: Seasonal
    
- 봄/여름/가을/겨울 사진 갤러리 + 계절별 오퍼 item들을 연결.
    

3. 스토리: 시소소를 시작한 이유
    

- tags: Story
    
- 호스트의 의도/철학, 공간의 이름 의미, 로고/컬러 톤 등 소개
    
- 이 스토리 하단에서 예약하기 CTA나 오퍼 보기로 이어짐.

## 5. subitems 사용 방식 요약

시소소 웹에서도, 너가 가진 구조 그대로 subitems를 이렇게 쓸 수 있음:

- 포함 내용
    
- 신청 오픈 / 신청 마감
    
- 유의 사항
    
- 환불 규정
    
- 하우스 룰
    
- 주변 추천 리스트 등
    

각 item(객실/오퍼/경험)에 필요한 세부 블록을 subitems로 쪼개서,  
프론트에서는 아코디언, 리스트, 박스 등으로 렌더링하면 됨.

## 1. 객실(ROOMS)

### 카테고리 문장 (공통 성질만)

이곳의 방에서는, 어디에 머물든 창을 통해 삼척 앞바다의 파도와 수평선을 한눈에 보게 됩니다.

---

### 방 1 · 바다 숨소리가 가까운 방

- 이 방은 파도가 바로 아래에서 부서지는 소리가 잘 들리는 층에 있습니다.
    
- 발코니에 서면 물이 닿는 암반과, 들고 나는 물길이 눈에 먼저 들어옵니다.
    
- 방 안에서는 침대 옆까지 파도 소리가 낮게 들어와, 밤에도 바다가 가까이 느껴집니다.
    

**기본 정보**

- 기준 2인
    
- 더블베드 1개
    
- 욕실 1개
    

---

### 방 2 · 수평선이 넓게 열린 방

- 이 방은 눈높이에서 수평선이 곧게 이어져 보이는 높은 층에 있습니다.
    
- 의자에 앉아서도 창 너머로 하늘과 바다가 만나는 선이 시야 대부분을 채웁니다.
    
- 해가 기울 때에는 빛이 방 안 깊숙이 들어와, 안에서 앉아서도 색이 바뀌는 하늘을 따라볼 수 있습니다.
    

**기본 정보**

- 기준 2인
    
- 트윈베드 2개
    
- 욕실 1개
    

---

## 2. 식사(FOOD)

### 카테고리 문장 (공통 성질만)

이곳의 식사는 삼척 앞바다와 주변 산에서 나는 제철 재료로, 그날 가져온 것에 맞춰 매일 조금씩 달라집니다.

---

### 봄 상차림

- 봄 상에서는 먼저 바다에서 막 건져 올린 부드러운 해조류와, 들녘에서 나온 연한 봄나물이 함께 올라옵니다.
    
- 초봄에는 파래·봄동 같은 잎채소가 상을 채우고, 늦봄에는 쑥과 어린 잎을 곁들인 따뜻한 밥과 국이 중심이 됩니다.
    

**예시 구성**

- 해조류 무침
    
- 봄나물과 된장국
    
- 그날 아침 시장에서 고른 생선구이
    

---

### 여름 상차림

- 여름 상에서는 입안을 식혀 줄 차가운 국과 얇게 썬 생선, 상에 시원함을 더해 주는 과일이 함께 나옵니다.
    
- 더운 날에는 얼음을 살짝 얹은 국과 산뜻한 채소가 먼저 나오고, 뒤이어 숯불에 구운 생선과 채소가 이어집니다.
    

**예시 구성**

- 차가운 오이·토마토 무침
    
- 얼음 위에 올린 회 한 접시
    
- 숯불 생선구이와 여름 채소
    
- 수박/참외 등 여름 과일
    

---

### 가을 상차림

- 가을 상에서는 갓 수확한 곡식과 기름이 오른 생선이 중심에 놓입니다.
    
- 밥상에는 흰쌀밥과 함께 밤·버섯이 들어간 솥밥이 돌아가며 올라오고, 은은하게 절인 장아찌와 김치가 곁을 잡습니다.
    

**예시 구성**

- 가을 채소 나물
    
- 등푸른 생선구이
    
- 햅쌀 솥밥 또는 버섯 솥밥
    

---

### 겨울 상차림

- 겨울 상에서는 몸을 먼저 데워 줄 뜨거운 국과, 미리 담가 둔 저장 음식들이 자리를 잡습니다.
    
- 따뜻한 국물 뒤에는 숯불에 구운 생선과 뿌리채소가 이어지고, 마지막에는 뜨거운 국에 밥을 말아 식사를 마무리합니다.
    

**예시 구성**

- 생선탕 또는 장국
    
- 숯불 생선·굴·대파·무
    
- 김치·장아찌 몇 가지
    
- 밥과 따뜻한 차

- 잎을 유리잔에 넣고 뜨거운 물을 부으면, 잔 안에서 색과 향이 천천히 퍼집니다.
    

**간단 정보**

- 기본 제공: 객실 내 티 세트
    
- 구매 가능: 로비 카운터 문의
    

---

- 원하는 분은 굽지 않은 채로 가져가도 되고, 조그만 불판에서 구워 맛을 볼 수도 있습니다.
    

**간단 정보**

- 구성: 김, 미역, 다시마 등
    
- 이용: 객실/로비에서 자유롭게 시식, 구매 가능
    

---

### 로비 아이템 · 공예품

- 로컬 공방에서 만든 잔·접시·나무 트레이와 소형 조명이 테이블 위에 섞여 있습니다.
    
- 객실에서 사용해 보고, 마음에 들면 체크아웃 시 골라 가져갈 수 있습니다.
    

**간단 정보**

- 종류: 도자기, 목공, 소형 조명 등
    
- 구매: 로비에서 작가·공방 정보 안내

**간단 정보**

## 1. 식사(FOOD)

### 1-1. 카테고리 오버뷰

이곳에서의 식사는, 정해진 이름의 코스가 아니라  
**그날 삼척 앞바다와 근처 산에서 어떤 재료가 도착했는가**로부터 시작됩니다.

아침마다 수산시장에서 고른 생선과 해조류,  
근방 농가에서 올라오는 채소와 곡식이  
봄·여름·가을·겨울, 절기마다 다른 상차림으로 올라옵니다.

메뉴판에는 복잡한 요리 이름 대신  
오늘 들어온 재료와 간단한 조리 방식이 적힙니다.  
손글씨 몇 줄만으로도, 그날의 공기와 온도가 함께 읽히는 식사를 준비합니다.

---

### 1-2. 카테고리 컨텐츠 (이미지 포함)

**섹션 제목:** 오늘의 상차림

**본문:**  
식당에 들어오면, 먼저 벽 한쪽에 걸린 작은 칠판을 보게 됩니다.  
그날 아침 바다와 시장, 농가에서 가져온 재료 이름들이  
날짜와 함께 적혀 있습니다.

“오늘의 바다: 도다리, 영덕 대게살, 생미역”  
“오늘의 산: 냉이, 달래, 감자, 쑥” 같은 식으로요.

상은 이 재료들을 중심으로  
과하지 않은 양과 단순한 조리법으로 채워집니다.  
한 끼가 끝난 뒤에도, 그날의 상은 기록으로만 남고  
다음 날이면 칠판의 목록이 또 바뀝니다.

**이미지 alt 텍스트(나중에 gallery로):**  
“칠판에 분필로 적힌 ‘오늘의 바다’, ‘오늘의 산’ 재료 목록과, 그 아래 소박하게 차려진 한 상의 상차림이 함께 보이는 장면”

---

### 1-3. 계절별 메뉴판 (각각 헤딩 + 리스트)

#### 봄 메뉴 – 해초와 봄나물

봄의 상은 차가운 바다에서 막 건져 올린 해초와,  
흙에서 막 올라온 연한 봄나물이 중심이 됩니다.

**예시 구성(리스트):**

- 봄 해초 모둠
    
    - 파래, 미역, 톳을 살짝 데쳐 간장과 유자즙으로만 무친 전채
        
- 들나물 된장국
    
    - 냉이와 달래를 넣어 끓인 묽은 된장국
        
- 진달래 화전
    
    - 찹쌀 반죽 위에 진달래 꽃잎을 올려 지진 작은 전병
        
- 봄 채소와 생선구이
    
    - 그날 시장에서 고른 생선을 숯불에 구워, 삶은 감자와 봄동과 함께 내는 메인
        
- 쑥밥
    
    - 쑥을 넣어 지은 밥에 소금과 들기름만 곁들인 마무리
        

---

#### 여름 메뉴 – 차가운 바다와 과일

여름의 상은 입안을 식혀 주는 **차가운 국과 생선**,  
그리고 달콤한 **여름 과일**에 기대어 펼쳐집니다.

**예시 구성:**

- 오이·토마토 냉국
    
    - 잘게 썬 오이와 토마토에 차게 식힌 다시국을 부은 국
        
- 얼음 위의 모둠회
    
    - 도미, 광어, 전갱이 등을 얇게 썰어 얼음 위에 올린 한 접시
        
- 해조류 샐러드
    
    - 미역, 다시마, 파래를 가볍게 절여 자두 드레싱과 함께 내는 샐러드
        
- 숯불 생선과 여름 채소
    
    - 작은 전어와 꽈리고추, 가지를 함께 구워내는 메인
        
- 참외·수박
    
    - 얇게 썬 참외와 수박, 소금을 살짝 찍어 먹는 디저트
        

---

#### 가을 메뉴 – 곡식과 기름 오른 생선

가을 상에는 갓 수확한 햅쌀과  
기름이 오른 등푸른 생선이 중심에 놓입니다.

**예시 구성:**

- 가을 나물 무침
    
    - 시금치, 취나물, 고사리를 가볍게 데쳐 들기름과 소금으로만 무친 나물
        
- 생선구이 두 가지
    
    - 고등어와 삼치를 굵은 소금에 재웠다가 숯불에 구운 메인
        
- 햅쌀 솥밥
    
    - 새로 도정한 쌀에 밤과 은행을 함께 넣어 지은 솥밥
        
- 된장지와 피클
    
    - 오래 숙성시킨 된장지 몇 가지와 가볍게 절인 채소
        
- 군고구마와 차
    
    - 껍질째 구운 고구마와 따뜻한 보리차
        

---

#### 겨울 메뉴 – 뜨거운 국과 저장 음식

겨울 상에서는 **뜨거운 국**과 미리 담가 둔 **저장 음식**들이  
차가운 바람을 대신 막아 줍니다.

**예시 구성:**

- 생선탕
    
    - 대구와 무, 파를 넣고 끓인 투명한 국물
        
- 김치와 장아찌
    
    - 김장김치, 깍두기, 무 피클 등을 작은 접시에 나눠 담은 반찬
        
- 숯불 굴·대구
    
    - 굴과 대구 살을 철망에 올려 구워, 초간장과 레몬으로만 간을 더한 메인
        
- 겨울 뿌리 채소 볶음
    
    - 우엉, 연근, 당근을 얇게 썰어 간장과 조청으로 볶은 반찬
        
- 뜨거운 다시국에 말은 밥
    
    - 마지막에는 다시국을 넉넉히 부어 밥을 말아 먹는 마무리

## 2. 로비(LOBBY)

### 2-1. 카테고리 오버뷰

이곳은 단지 한 채의 숙소가 아니라,  
**이 동네에서 오래 살아온 사람들과 함께 만드는 곳**입니다.

우리는 매 시즌, 로컬 파트너들과 이야기를 나누고  
지금 이 시기에 이 동네를 가장 잘 보여 줄 수 있는 것들을 함께 고릅니다.

로비는 그 관계가 가장 먼저 눈에 보이는 자리입니다.  
바다와 산, 마을에서 온 물건들이  
“이런 것들을 소개하고 싶다”는 마음과 함께 놓여 있습니다.

해마다, 때로는 달마다  
우리가 지금 소개하고 싶은 것들을 새로 골라  
로비 테이블 위에 올려 둡니다.

---

### 2-2. 로비 카테고리 컨텐츠 (이미지 포함)

**섹션 제목:** 로컬 파트너들의 작은 진열대

**본문:**  
입구를 지나 로비로 들어오면,  
눈높이의 유리 진열장이 아니라  
허리 높이의 긴 테이블이 먼저 보입니다.

그 위에는 이 동네 어부가 말려 온 해조류,  
산자락에서 약초를 채집하는 이가 만든 티,  
골목 공방에서 구운 잔과 접시가 나란히 놓여 있습니다.

각 물건 옆에는 짧은 메모가 붙어 있습니다.  
“○○항, 김○○ 어부의 올해 첫 김”  
“△△마을, 이○○ 님이 봄에 채집한 잎으로 만든 블렌드 티”처럼요.

머무는 동안 이 물건들은  
‘전시품이 아닌, 함께 써 볼 수 있는 것’으로 남아 있습니다.

**이미지 alt 텍스트:**  
“나무 테이블 위에 해조류 묶음, 약초 티 병, 손으로 빚은 잔과 접시가 섞여 놓여 있고, 각각 옆에 손글씨 메모가 붙어 있는 로비 풍경”

---

### 2-3. 로비 아이템 예시

#### 로비 아이템 · 티

- 이 동네 산자락에서 말린 잎과 약초로 만든 블렌드 티가 투명한 병에 담겨 있습니다.
    
- 뚜껑을 열어 향을 맡아 볼 수 있고, 객실 안에서도 마음에 드는 블렌드를 골라 우려 마실 수 있습니다.
    

**간단 정보**

- 구성 예: 감잎·쑥 블렌드 / 솔잎·구절초 블렌드 등
    
- 이용: 객실 내 티 세트 제공, 추가 구입 가능
    

---

#### 로비 아이템 · 해조류

- 항구 근처에서 말린 김, 미역, 다시마가 묶음 단위로 정리되어 있습니다.
    
- 작은 인덕션과 팬이 함께 비치되어 있어, 즉석에서 구워 맛을 볼 수도 있습니다.
    

**간단 정보**

- 구성 예: 김 10장, 미역 1봉, 다시마 조각 3~4장
    
- 이용: 시식 후 원하면 포장 구매 가능
    

---

#### 로비 아이템 · 공예품

- 머그잔, 찻잔, 작은 접시, 나무 트레이와 조명 스탠드가 테이블 한쪽에 놓여 있습니다.
    
- 객실로 가져가 사용해보고, 체크아웃 시 마음에 든 것을 선택해 가져갈 수 있도록 되어 있습니다.
    

**간단 정보**

- 참여 공방 예: 그릇 공방 1곳, 목공 작업실 1곳, 조명 작가 1명
    
- 구매: 로비에서 각 작가·공방의 연락처와 가격표 안내

## 3. 체험(EXPERIENCES)

### 3-1. 카테고리 오버뷰

로비에서 만난 물건들이  
이 동네의 사람과 장소를 가리킨다면,

체험 프로그램은 그 사람들과 함께  
**직접 그 장소를 걸어 보는 일정**입니다.

어부의 김 묶음을 로비에서 봤다면,  
체험에서는 그 어부가 새벽에 나가는 포구를 함께 걷습니다.

약초 티 병 옆의 이름을 읽었다면,  
체험에서는 그 약초가 자라는 산자락을 함께 오릅니다.

이렇게 로컬 파트너들과의 관계가  
로비 → 체험으로 자연스럽게 이어지도록 짜여 있습니다.

---

### 3-2. 체험 카테고리 컨텐츠 (이미지 포함)

**섹션 제목:** 이 동네에서 보내는 하루

**본문:**  
체험들은 따로 떼어 보면 각각의 프로그램이지만,  
하루를 통째로 이 동네에 맡겨보면 하나의 흐름을 이룹니다.

새벽에는 바닷가를 걷고,  
아침에는 수산시장에서 오늘의 재료를 보고,  
낮에는 공방이나 산길을 찾았다가,  
저녁에는 모닥불 옆에 앉아 하루를 마무리합니다.

각 체험은 로비에서 만난 이름과 연결되어 있어,  
스크롤을 내리며 읽기만 해도

### 구현 관점에서 한 번만 정리해보면

- **동적 바인딩 최소화**
    
    - {variable} 형태는 전부 뺐고,
        
    - CTA도 예약하기, 자세히 보기, 문의처럼  
        카테고리 공통으로 써도 어색하지 않게 넣어둔 상태야.
        
- **Airtable / Framer 연동**
    
    - tag0 → 섹션 단위 데이터 소스
        
    - items → 각 섹션 안에 들어갈 블록
        
    - subitems → 이웃들 웹사이트 링크처럼, 블록에 붙는 서브 정보
        
    - JSON 배열(["id1","id2"])는 지금처럼 문자열로 두고,  
        Framer에서 파싱해서 관계 매핑하면 돼.
        

이제:

- 실제 숙소 이름/실제 파트너 이름/실제 URL만 정해지면  
    그 부분만 찾아 바꾸기 하면 바로 프로덕션에 올릴 수 있는 수준이라,
    
- 필요하면 다음 단계로,
    
    - 맵핑용 Framer 구조(컴포넌트 props 구조)
        
    - Airtable view 설계(예: db별 필터, tag0별 필터)  
        같은 것도 같이 짜 줄 수 있어.

### 설명 간단히 정리

- version: PROTOCOL의 "version": "1"과 맞춤
    
- job: 이 작업 폴더 이름으로 쓸 문자열 (예시: "local-stay-web")
    
- base: Airtable에서 쓸 베이스 이름 (예시: "Local Stay Web")
    
- tables
    
    - tag0
        
        - CSV: new_tag0.csv
            
        - _record_id 기준으로 items.items, items.tag0와 링크
            
        - items 필드: LNK → items 테이블
            
        - stories 필드: 스토리를 따로 안 쓰고 있지만, 구조 맞춰 items로 링크 가능하게 열어둔 상태
            
    - items
        
        - CSV: new_items.csv
            
        - tag0: LNK → tag0 (섹션–아이템 관계)
            
        - subitems: LNK → subitems (특히 이웃들 링크용)
            
        - db, layout은 SEL로 두어서 Framer 컴포넌트 타입 스위칭할 때 쓰기 좋게 함
            
    - subitems
        
        - CSV: new_subitems.csv
            
        - items: LNK → items (어떤 item에 붙는 링크인지)
            
        - notes: 링크 텍스트
            
        - notes2: URL

이제 구조상:

- Airtable에 tag0 / items / subitems 테이블 만들고
    
- 각자 CSV import
    
- SCHEMA.json(PROTOCOL) 기반으로 링크 필드만 설정해주면  
    Framer에서
    
- tag0로 페이지 섹션 루프
    
- 각 tag0.items 관계로 items 렌더
    
- items.subitems로 이웃들 웹사이트 링크까지 같이 뿌리는 구조가 깔끔하게 맞을 거야.
    

여기까지 세팅해 본 다음에,  
프레이머에서 실제로 **한 섹션(예: 식사)**를 바인딩해 보면서

- 더 잘게 쪼개고 싶은 필드
    
- 버튼 텍스트/CTA를 컴포넌트화하는 방식  
    같은 걸 다음 단계에서 같이 맞춰보면 좋을 것 같아.

안에 들어있는 태그는 이렇게 구성돼:

    - items: food_intro, food_menu_spring, food_menu_summer, food_menu_autumn, food_menu_winter
        
2. 계절 태그
    
    - tag_spring / spring / 봄 / db: season → food_menu_spring
        
    - tag_summer / summer / 여름 / db: season → food_menu_summer
        
    - tag_autumn / autumn / 가을 / db: season → food_menu_autumn
        
    - tag_winter / winter / 겨울 / db: season → food_menu_winter
        
3. 카테고리 태그
    
    - tag_shop / shop / 샵 / db: category
        
        - items: lobby_item_seaweed, lobby_item_tea, lobby_item_tray
            
    - tag_experience / experience / 체험 / db: category
        
        - items: exp_dawn_sea, exp_market_walk, exp_evening_fire
            
    - tag_neighborhood / neighborhood / 이웃 / db: category

## 2) SCHEMA에 tags 테이블까지 추가하고 싶다면

아까 준 SCHEMA.json에 이 블록만 추가하면 돼:

Json

`{   "name": "tags",  "csv": "new_tags.csv",  "primary_key": "_record_id",  "fields": {    "_record_id": "TXT",    "name": "TXT",    "labelKr": "TXT",    "db": "SEL",    "order": "TXT",    "items": {      "type": "LNK",      "target": "items"    },    "stories": {      "type": "LNK",      "target": "items"    }  } }`

그러면:

- 태그별로 어떤 item들이 묶여 있는지
    
- 프레이머에서 태그 필터(봄/여름/가을/겨울, Food/Shop/Experience/Neighborhood)  
    같은 것도 데이터만으로 돌릴 수 있어.

- new_tag0.csv

- 실제 Airtable에 이 4개 올리고

    - 태그 필터(봄/여름/가을/겨울)
        
    - 카테고리별 리스트(예: Shop, Experience)  
        어디까지 자동화할지 같이 정리해보면 좋을 것 같아.
