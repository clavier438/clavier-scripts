## PART 1: 단 하나의 필터 (The One Filter)

### 1.1 내 옵시디언 원칙이 이미 답

```
"독자 상품을 주력으로, 통상적 상품은 백업으로"
"OTA가 놓친것 = 중점배치, 기본사항 = 백업 배치"
```

**이것을 웹사이트 요소에 적용:**

```
질문: "이 요소가 니치의 압도적 차별화를 강화하는가?"

YES → 메인 배치 (Hero~3번째 섹션)
NO → 백업 배치 (Footer) 또는 제거
```

```
YES (독특한 구조):

NO (표준 룸):
- 제거 (사진이 더 효과적)
```

```
HOME
```

## PART 5: 작업 전 체크리스트 (이것만 확인)

### 5.1 요소 추가 전 질문 (3개)

```
[ ] 1. 이것이 우리 니치의 압도적 차별화를 강화하는가?
       → NO면 다음 질문

[ ] 2. 이것이 OTA에 없는 독점 정보인가?
       → NO면 다음 질문

[ ] 3. 이것이 예약 경로를 방해하지 않는가?
       → NO면 제거
```

### 5.2 실전 워크플로우

```
Step 1: 니치 정의 (1문장)
"우리는 [건축 철학]으로 차별화된 호텔이다"

Step 2: 압도적 요인 (1개)
"공간의 철학"

Step 3: 요소 필터링
├── Philosophy/Design → 메인 (압도적 요인)
├── Floor Plan → 메인 (OTA 독점)
├── Gallery → 메인 (시각 증명)
├── Rooms → 메인 (필수)
├── Offers → 서브 (보너스)
├── SNS Feed → 제거 (방해)
├── Testimonials → 제거 (무관)
```

## 마지막 조언: "장바구니 유혹" 극복법

```
다 담고 싶을 때:

"이게 없어도 우리 니치가 명확한가?"
└─ YES → 제거
└─ NO → 유지

"이게 있으면 예약이 빨라지는가?"
└─ NO → 제거
└─ YES → 유지

"OTA에서 이미 본 정보인가?"
└─ YES → 제거 (중복)
└─ NO → 유지 (독점)
```

- 메인 요소: 4-6개 (더 이상 NO)
- 서브 요소: 2-3개
- Footer: 나머지
