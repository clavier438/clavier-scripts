## 3-6. 타깃 리스트/계약서/콜드 메일 자료 등 완비 (11/12~14) — 실무 상세 가이드

### 1️⃣ 타깃 클라이언트 리스트 정리 (20개)

당신이 이미 확보한 20개 숙박시설 리스트를 **체계적으로 정리**합니다.

#### A. 엑셀/구글시트 구조 예시

| # | 상호명 | 지역 | 객실수 | 연락처(전화) | 이메일 | 카톡ID | 웹사이트 | OTA | 우선순위 | 상태 | 비고 |
|----|--------|------|--------|----------|--------|--------|---------|-----|----------|------|------|
| 1 | OO게스트하우스 | 제주 | 6 | 010-1234-5678 | oo@naver.com | oo_guesthouse | naver.com/... | O | A | 미발송 | 직연락처 확인 |
| 2 | XX호텔 | 부산 | 8 | 051-123-4567 | - | xx_hotel | - | O | B | 미발송 | 이메일만 있음 |
| 3 | ...| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |

#### B. 각 항목 설명

| 컬럼 | 설명 | 활용 |
|------|------|------|
| # 및 상호명 | 순번, 숙소 이름 | 빠른 참조 |
| **지역** | 제주, 부산, 강원 등 | 지역별 특성 파악 |
| **객실수** | 5~10실 범위 확인 | 타깃 필터링 |
| **연락처(전화)** | 직통 번호 (숙소 또는 오너) | 전화 영업용 |
| **이메일** | 예약 또는 비즈니스 메일 | 이메일 발송용 |
| **카톡ID** | 카톡 오픈채팅 또는 ID | 카톡 메시지 발송용 |
| **웹사이트** | 기존 사이트 URL (있으면) | 현황 분석 |
| **OTA** | Airbnb, Booking.com 사용 여부 | 니즈 파악 (OTA 의존도) |
| **우선순위** | A(즉시), B(중기), C(후순위) | 영업 순서 결정 |
| **상태** | 미발송, 발송, 응답 대기, 상담 중, 계약, 거절 등 | 진행상황 추적 |
| **비고** | 특이사항, 메모 | 맞춤 대응 |

#### C. 우선순위 선정 기준

- **A 우선**: 직연락처 확보, OTA 적극 사용, 웹사이트 없거나 구식
- **B 중순**: 연락처는 있지만 확인 어려움, OTA 사용 중간
- **C 후순**: 정보 부족, 이미 최신 웹사이트 있음

#### D. 리스트 관리 팁

- **구글시트** 추천 (실시간 공동 수정, 모바일 접근 용이)
- 매일 아침 "상태" 컬럼 업데이트 (발송, 응답 대기 등)
- 필터 기능으로 "아직 발송 안 한 것" 빠르게 확인
- 조건부 서식: "계약" 행 초록색, "거절" 행 회색으로 시각화
