## 성과 측정 시스템 제안: 최소한의 툴로 4가지 지표 추적

당신의 상황을 정리하면:

- **Framer** (웹사이트)
- **Onda PMS** (예약/객실 관리)
- 4가지 성과 지표 추적 필요
    1. 숙박수익 증가분
    2. 직접예약 증가로 인한 OTA 수수료 절감분
    3. 부가수익 증가분
    4. 리드풀 증가분 (DCF 10-12%, 3-5년)


### 제안: **Google Sheets + 수동 입력 (월 1회)**

**이유:**

1. **가장 심플한 구조**
    - 복잡한 통합 없음
    - 월 1회 중간정산 때 클라이언트와 함께 수치 입력
    - 계산 공식은 자동화되지만, 데이터는 수동
2. **실제로 필요한 것**
    - Onda에서 월별 데이터 추출 (CSV 또는 직접 확인)
    - OTA별 예약 건수와 수수료율
    - 부가수익 (현금 or 별도 장부)
    - 리드풀 지표 (뉴스레터 구독자, SNS 팔로워, 문의 건수)
3. **Google Sheets가 최선인 이유**
    - 무료
    - 공유 쉬움 (클라이언트와 실시간 확인)
    - DCF 계산 공식 구현 가능
    - 차트 자동 생성
    - 버전 히스토리 (변경 추적)
### 구체적 구조

**시트 구성:**

**Sheet 1: 월별 원본 데이터 입력**

```
날짜 | 총 예약건수 | 직접예약 | OTA예약 | OTA수수료율 | 숙박매출 | 부가수익(F&B) | 부가수익(굿즈) | 부가수익(기타) | 뉴스레터 | SNS팔로워 | 문의건수
```

**Sheet 2: 자동 계산 (성과 측정)**

```
= Before (계약 전 3개월 평균 or 첫 달 기준)
= After (매월 누적)
= 차이값 자동 계산

1. 숙박수익 증가분
2. OTA 수수료 절감분 = (직접예약 증가 건수 × 평균 객실료 × 평균 OTA 수수료율)
3. 부가수익 증가분
4. 리드풀 미래가치 = DCF(리드풀 증가분, 할인율 10-12%, 3-5년)
```

**Sheet 3: 대시보드 (시각화)**

- 월별 트렌드 라인 차트
- 4가지 성과 지표 바 차트
- 총 성과 누적 (당신 보수 자동 계산 10%)
### DCF 공식 예시 (Google Sheets)

리드풀의 미래가치를 환산하는 공식:

```
= (리드당 예상 평균 수익 × 리드풀 증가분 × 컨버전율) / (1 + 할인율)^년차
```

예시:

- 리드풀 100명 증가
- 리드당 예상 평균 수익: $500 (3-5년간 1번 예약 가정)
- 컨버전율: 20%
- 할인율: 12%
- 기간: 3년

```
Year 1: (500 × 100 × 0.2) / (1.12)^1 = $8,928
Year 2: (500 × 100 × 0.2) / (1.12)^2 = $7,971
Year 3: (500 × 100 × 0.2) / (1.12)^3 = $7,117
총 현재가치: $24,016
```

### 실행 플로우

**계약 전:**

1. 클라이언트와 Google Sheets 템플릿 공유
2. Before 데이터 입력 (3개월 평균 or 계약 첫 달 기준)

**매월 (중간정산 시):**

1. Onda에서 숙박 데이터 확인 (5분)
2. 부가수익 현금 장부 확인 (3분)
3. 리드풀 지표 확인 (뉴스레터, SNS, 문의) (2분)
4. Google Sheets에 입력 (5분)
5. 자동 계산된 성과 확인 → 보수 청구

**총 소요 시간: 15분/월**

### 왜 Airtable, Notion, 자동화 툴을 제안하지 않는가?

**Airtable:**

- 장점: 예쁘고, 관계형 데이터베이스
- 단점: **과잉**. 당신은 매월 1회만 데이터 입력하면 됨. Airtable의 자동화, 뷰, 필터 기능은 필요 없음.

**Notion:**

- 장점: 문서 + 데이터베이스
- 단점: 계산 공식이 Sheets보다 약함. DCF 같은 복잡한 재무 계산엔 부적합.

**자동화 (Zapier, Make 등):**

- Onda → Sheets 자동 연동?
- 문제: Onda API가 있어도, 부가수익과 리드풀은 여전히 수동 입력 필요.
- 부가수익은 클라이언트가 별도 관리 (현금, 간이 영수증)
- 리드풀은 SNS, 뉴스레터, 전화 문의 등 여러 소스

**결론:** 자동화 구축 시간 > 월 1회 수동 입력 시간
