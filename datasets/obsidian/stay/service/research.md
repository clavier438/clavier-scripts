## 공통 패턴 5가지

1. **압도적 니치 차별화** → 가족만, 그린만, 로맨틱만 → OTA 가격 비교에서 완전 이탈
2. **웹사이트 = 예약 허브** → Depth 2~3, sticky booking bar → Offers 탭 네비 최상위
3. **Exclusive 혜택 OTA 미제공** → 직접 예약 시에만 업그레이드/패키지/할인
4. **AI/테크 적극 활용** → dynamic pricing, personalization → 전환율 2~3배
5. **재방문률 40% 이상** → 니치 + 독점 경험 + 개인화 → 충성 고객 → OTA 불필요

---

## Shallow IA 구조 근거

**직접예약률 80~100% 호텔의 85%가 Depth 2~3 유지**

인과 체인:
```
Shallow IA (Depth 2~3)
→ 클릭 2~3번 예약 도달
→ OTA(6~8클릭) 대비 빠름
→ Exclusive 혜택 가시성
→ 가격 비교 전 전환
→ 직접예약 80~90%
→ OTA 수수료 절감 15~30%
→ 수익 증가 20~30%
```

전환율: 일반 호텔 2~3% → 극강 사례 5~8% (2~3배)

## 섹션 요소 판단 3단계 필터

```
Q1. 이것이 니치의 압도적 차별화를 직접 강화하는가?
    YES → 메인 배치 (Depth 1~2)
    NO  → Q2

Q2. 이것이 OTA에 없는 독점 정보인가?
    YES → 서브 배치 (Depth 2~3)
    NO  → Q3

Q3. 이것이 예약 경로를 방해하는가?
    YES → 제거
    NO  → Footer
```

**니치별 메인 요소 수:** 4~6개 이상 NO

## Exclusive 혜택 배치 원칙

- **극단적 니치 (건축, 가족 전용)** → 나중에 배치 (4번째) — 니치 자체가 Exclusive
- **경쟁 있는 니치 (로맨틱, 웰니스)** → 앞에 배치 (2번째) — OTA 비교 전 차별화 필요
- **Sticky CTA "Book Direct - Best Rates"** → 항상, 모든 극강 사례 100% 적용
