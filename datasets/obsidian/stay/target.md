# 부티크 호텔 니치 공략용 도요타식 칸반 운영 정리 (Trello + HubSpot)

## 1. 현재 비즈니스 목표 정리

**· 근무 시간·수입 목표**

- 오전 4시간만 집중해서 일하기
- 월 500(만 원 기준으로 가정) 이상 안정적으로 벌 수 있는 구조 만들기

**· 비즈니스 모델(큰 그림)**

- 같은 니치, 같은 문제, 같은 기획, 같은 구조를 여러 곳에 반복 적용하는 모델
- "한 번 잘 만든 구조를 여러 번 되파는" 방식
- 그 구조를 담는 수단이 **웹사이트(사이트 제작·세팅·운영)**
- 첫 니치: **부티크 호텔** (소규모/개성 있는 호텔들을 대상으로 동일한 문제를 해결해 주는 서비스)

---

## 2. 칸반/도요타식 관점에서 본 당신의 비즈니스

**· 작업 흐름(Flow) 관점에서 보는 단계**

1. 니치 리서치 & 포지셔닝

   - 부티크 호텔 시장 조사
   - 공통으로 가지고 있는 문제 정의 (예: 직예약 부족, 브랜딩 약함, 모바일 경험 나쁨 등)

2. 공통 기획·템플릿 설계

   - 사이트 정보 구조(섹션 구성)
   - 카피 구조(헤드라인, 설명, 예약 유도 문구 등)
   - 기능 세트(예약 버튼, 문의 폼, 지도, 리뷰 섹션 등)
   - "언제든 여러 호텔에 복붙 가능한 기본 틀"을 만드는 단계

3. 개별 호텔 리스트업 (리드 발굴)

   - 부티크 호텔 후보들을 찾고 정리
   - 이 리스트는 **HubSpot에서 관리** (클라이언트·잠재 고객 리스트)

4. 컨택·영업

   - 이메일, 전화, DM 등으로 첫 접촉
   - 문제·니즈 파악, 서비스 설명, 가격 제안

5. 계약·세팅 준비

   - 조건 합의, 계약, 입금
   - 로고, 사진, 텍스트 등 사이트 제작에 필요한 자료 수집

6. 사이트 커스터마이징 & 제작

   - 공통 템플릿에 호텔별 정보만 끼워 넣기
   - 필요한 경우 소규모 커스터마이징

7. 런칭 & 운영

   - 사이트 공개
   - 기본 모니터링(폼 작동 여부, 예약/문의 흐름 등)

8. 유지보수 & 업셀

   - 소규모 수정, 시즌별 컨텐츠 수정 등
   - 광고·SEO·추가 기능 등 업셀 포인트 제안

**→ 이 전체 흐름을 칸반보드(트렐로) 위에 그대로 올려서 시각화하고 관리하는 게 핵심.**

## 3. 도요타식 칸반의 핵심 포인트 요약

**1) "할 일 목록 관리"가 아니라 "흐름 관리"**

- 보드는 단순 To-Do 리스트가 아니라, 일이 어디서 막히는지(병목)를 찾기 위한 도구
- 질문 방향:
  - "무엇이 남았지?"보다
  - "어디에 카드가 쌓여 있고, 왜 거기서 멈췄지?"에 집중

**2) Pull(당겨오기) 시스템**

- 새 작업을 위에서 억지로 밀어 넣는 Push 방식이 아니라,
  - 각 단계 담당자가 **자기 칼럼에 여유가 생겼을 때 다음 카드를 당겨오는 방식**
- 예시:
  - 오전 작업 시작: 내가 오늘 "제작"을 담당한다면, 제작 칼럼에 빈 자리가 있을 때만
    - 이전 단계(계약 완료/자료 수집 완료 칼럼)에서 카드를 하나 가져옴

**3) WIP Limit(동시 진행 작업 개수 제한)**

- 각 칼럼에 "동시에 올려둘 수 있는 카드 수"를 명시적으로 제한
- 예:
  - 상담 중: 최대 2개
  - 계약 진행: 최대 3개
  - 제작 중: 최대 2개
- 규칙:
  - 칼럼 카드 수가 한도(WIP)를 넘으면 새로운 작업을 그 칼럼으로 절대 옮기지 않음
  - 먼저 "왜 여기서 막히는지"를 확인하고 원인 해결부터

**4) 카드 = 작업 지시서 + 문제 탐지 장치**

- 각 카드(호텔 1곳)에 들어가야 할 최소 정보
  - 호텔 이름, 위치
  - 담당자·연락처
  - 현재 단계
  - **다음 액션 1개(구체적)**
  - 예상/실제 금액(구축비 + 월 유지비)
  - 시작일·런칭일(리드타임 확인용)

**5) 정기적인 보드 리뷰 = 카이젠(지속 개선)**

- 매일/매주 짧게 보드를 보면서:
  - 특정 칼럼에 카드가 계속 쌓여 있는지
  - 특정 유형의 작업이 유독 오래 걸리는지
  - WIP Limit를 자주 넘는 구간이 있는지
- 그에 맞춰
  - 칼럼 구조 조정
  - WIP 숫자 조정
  - 템플릿·프로세스 개선\
    을 반복하는 구조

## 4. 트렐로 보드 구조 초안 (부티크 호텔 니치 전용)

트렐로에서 쓸 수 있는 **칼럼(리스트) 구조 예시**입니다.

1. 아이디어 / 타깃 후보

- 아직 조사 전인 호텔들
- 간단한 정보만 적어둔 상태 (이름 / 위치 / 웹사이트 링크 정도)

2. 리서치 완료

- "내가 타깃으로 삼을 가치가 있다"고 판단 끝난 호텔 카드들
- 문제·특징 간단 메모

3. 컨택 준비

- 맞춤 제안 준비 중인 호텔들
- 카드에 넣어둘 것:
  - 사용할 이메일/메시지 템플릿 종류
  - 제안 포인트 메모

4. 컨택 발송

- 실제로 이메일/DM 등을 보냈고, 답을 기다리는 상태
- 카드에 마지막 발송일 기록

5. 상담 중

- 콜/미팅 등으로 대화가 오가고 있는 단계
- 카드에 다음 액션(예: "견적안 보내기", "자료 요청하기") 명확히 한 줄로 적기

6. 계약 진행

- 조건 합의, 계약서, 입금, 자료 수집 등 실제로 돈과 자료가 오가는 단계
- 체크리스트 예:
  - 계약서 송부/서명
  - 선금 입금
  - 로고/사진/텍스트 수령

7. 제작 중 (사이트 세팅)

- 템플릿에 정보 입력, 기본 디자인 조정 등 작업 진행 중
- WIP Limit를 강하게 걸어두기 좋은 칼럼 (예: 최대 2개)

8. 런칭 완료

- 실제로 사이트가 공개된 상태
- 카드에 런칭일 기록

9. 운영 & 업셀

- 유지보수 중인 호텔들
- 정기 점검, 업셀 아이디어 메모(광고, 사진 리뉴얼, 다국어 버전 등)

## 5. Trello + HubSpot 역할 정리

**· Trello (칸반/작업 흐름 관리)**

- 각 호텔마다 카드 1장
- 단계(리스트)를 옮겨가며 상태 추적
- WIP Limit를 적용해 "동시에 하는 일의 개수"를 제어
- 오전 4시간 동안 무슨 일을 할지 바로 보드에서 결정

**· HubSpot (클라이언트·리드 정보 관리)**

- 호텔별 상세 정보, 연락 이력, 세일즈 파이프라인 정보 저장
- 이메일 발송·트래킹, 디테일한 CRM 기능 담당
- Trello 카드에는 요약만, HubSpot에는 상세 정보

**이렇게 역할을 나누면:**

- Trello: "지금 어떤 호텔이 어떤 단계에 있고, 다음으로 뭘 해야 하는가?"를 한눈에 보는 도구
- HubSpot: "이 호텔과는 언제 어떤 대화를 했고, 매출·기회는 얼마인가?"를 관리하는 도구

## 6. 오전 4시간 근무를 위한 기본 원칙

1. 보드 먼저 보기

- 하루 시작할 때 Trello 보드부터 확인
- 각 칼럼의 카드 수와 WIP Limit 비교해서
  - "어디가 막혀 있는지"를 우선 파악

2. 병목부터 해결

- 새 작업을 시작하기 전에
  - WIP Limit를 넘겼거나
  - 카드가 오래 멈춘 칼럼의 일을 먼저 처리

3. Pull 원칙 지키기

- 내 칼럼에 빈자리가 생겼을 때만
  - 이전 칼럼에서 카드를 가져와서 당겨오기
- 절대 한 번에 여러 칼럼에 카드를 우겨 넣지 않기

4. 작은 단위로 끊어서 끝내기

- 하나의 카드(호텔) 안에서도 작업을 작게 쪼개어
  - 오전 4시간 내에 "끝냈다"고 말할 수 있는 단위를 명확히 하기

## 7. 다음 단계 제안

- 이 문서를 기준으로:

  1. Trello 실제 보드에 위 구조를 반영해서 리스트를 만들고
  2. 각 리스트별 WIP Limit 숫자를 정한 뒤
  3. HubSpot에 이미 있는 클라이언트/리드 중 부티크 호텔 후보들을 Trello 카드로 옮겨오는 작업

- 이후에는

  - "실제 Trello 보드 스크린샷" 또는
  - "각 리스트에 현재 카드(호텔)가 몇 개 있는지"를 알려주시면
  - WIP 숫자 조정, 오전 4시간 구체적인 루틴(예: 1시간 컨택, 1시간 상담 준비, 2시간 제작 등)까지 더 미세하게 설계해 드릴 수 있습니다.
