## 혁 님 상황에 맞는 최종 추천

혁 님이 하려는 것의 본질은 **"의미를 파는 콘텐츠 퍼스트 브랜드"**입니다. 이걸 기준으로 보면:

**지금 바로 시작한다면 → Write (무료)**로 뉴스레터/블로그를 먼저 런칭해서 Build in Public 시작.

**본격적으로 구축한다면 → Sharp**가 정답입니다. 포트폴리오는 당신이 만든 것을 보여줍니다. 케이스스터디 포트폴리오는 당신이 만든 것을 어떻게, 왜 만들었는지를 설명합니다. 이것이 바로 "상품이 아니라 의미를 파는" 웹사이트의 구조입니다.

**퍼스널 브랜드 + 상품 판매를 동시에 → Creator Kit**으로 스토리와 커머스를 한 곳에.





# 오프라인 비즈니스

포트폴리오 말고 **오프라인 비즈니스용인데, 우리가 말한 ‘의미/콘텐츠 중심’ 철학에 맞는 웹사이트 타입**을 찾는 거라면, 방향을 조금만 튜닝하면 충분히 가능합니다.

### 1. 어떤 오프라인 비즈니스에 쓰려고 하는지 먼저 정리

오프라인이라고 해도 결이 많이 달라서, 템플릿 선택 기준이 좀 달라집니다.

- 동네 기반: 카페, 바, 레스토랑, 바버샵, 플라워샵
- 예약 기반 서비스: 요가/피트니스 스튜디오, 미용실, 테라피/상담, 클리닉
- 공간/경험: 전시 공간, 팝업 스토어, 워크숍/클래스, 코워킹

혹시 생각하고 있는 업종이 이미 있으면 말해주면, 그 업종에 딱 맞는 구조를 골라줄 수 있어요.

---

### 2. “오프라인 비즈니스 + 의미/콘텐츠 중심”일 때 템플릿 고르는 기준

Framer 마켓플레이스에서 템플릿을 볼 때, **포트폴리오가 아닌 로컬/비즈니스용인데도 콘텐츠를 밀어 넣을 수 있는지**가 핵심입니다.

특히 이런 구조를 가진 템플릿을 찾으면 좋아요:

1. **Hero에 스토리·태도 공간이 넓은 것**
    
    - 첫 화면에 “메뉴/가격표”만 있는 템플릿 말고 
    - 브랜드 스토리, 마인드셋, 철학 한 단락을 크게 넣을 수 있는 레이아웃
2. **Story 섹션이나 Journal/Blog 블록이 기본 포함된 것**
    
    - Restaurant / Local Business 템플릿 중에서도 
    - “Our Story”, “Journal”, “News & Events” 같은 페이지/섹션이 들어간 것  
        → 여기서 빌드인퍼블릭스럽게 “왜 이 공간을 이렇게 운영하는지” 계속 쌓을 수 있음
3. **메뉴·서비스 리스트가 “설명”을 강조하는 스타일**
    
    - 단순히 “서비스명 + 가격” 리스트가 아니라 
    - 각 서비스/메뉴 아래에 설명 텍스트 길게 넣어도 깨지지 않는 타입  
        → “이 서비스가 어떤 상징/경험인지”를 써 넣기 좋음
4. **예약(Booking)이나 문의 CTA가 자연스럽게 얹힌 것**
    
    - 스토리 읽은 다음에 바로 
    - “방문 예약 / 워크숍 신청 / 자리 문의”로 이어지게 하는 버튼 구조
5. **사진·공간 이미지를 크게 보여주는 블록**
    
    - 오프라인 비즈니스라면 결국 “공간의 느낌”이 중요하니까 
    - 풀스크린 이미지 + 짧은 카피 조합 섹션이 있는 템플릿이 유리

---

### 3. Framer에서 실제로 어떻게 찾으면 좋냐면

Framer 템플릿 마켓(Templates)에서 카테고리·검색을 이렇게 쓰면 돼요:

1. 카테고리 쪽에서 
    - `Business` / `Local Business` / `Restaurant` / `Service` / `Studio` 같은 걸 먼저 선택
2. 검색 키워드에 
    - `story` `journal` `blog` `studio` `restaurant` `local` `salon` `wellness` 같은 단어를 조합해보기

그중에서 **썸네일만 봐도**:

- 텍스트 블록이 많고
- 섹션이 “스토리 → 서비스/메뉴 → 후기/저널 → 예약/문의” 흐름으로 가는 템플릿이 후보입니다.

---

### 4. 오프라인 비즈니스에 잘 맞는 구조 예시 2가지

혁 님이 어떤 업종을 해도, 이 두 가지 구조 중 하나로 맞출 수 있습니다.

#### A. 동네 기반 공간(카페, 샵, 바, 갤러리 등)

구조 예:

- Hero: “우리가 이 공간을 왜 여는지” 한 문장 + 짧은 스토리 
- Section 1: 공간 사진 + “공간 철학”(음악, 조명, 책, 동선 등) 
- Section 2: 메뉴/상품 리스트 + 각 항목에 스토리 한 줄씩 
- Section 3: Journal/News – 
    - 새로운 시즌 메뉴 이야기 
    - 콜라보, 이벤트, 워크숍 기록
- Section 4: 방문 정보 + 예약/문의 버튼

이 구조에 맞는 템플릿은 Framer에서 

- “Cafe / Restaurant / Bar / Local shop” 계열 템플릿 중 
- Journal/News 페이지 포함된 걸 고르면 바로 쓸 수 있어요.

#### B. 예약 기반 서비스(스튜디오, 상담, 살롱 등)

구조 예:

- Hero: “내가 하는 일의 의미”를 한 문장으로 (ex. “불안한 몸을 다시 집처럼 느끼도록 돕습니다”) 
- Section 1: 서비스/세션 타입 소개 + 각각의 “의도” 설명 
- Section 2: Process/How it works – 
    - 첫 방문 시 어떤 경험을 하게 되는지
- Section 3: Stories/Voices – 
    - 후기, 혹은 클라이언트 케이스의 내러티브
- Section 4: Journal/Notes – 
    - 혁 님의 생각, 에세이, 빌드인퍼블릭 느낌 글
- Section 5: 예약 폼 / 캘린더

여기에는 `Studio / Wellness / Coaching / Consultant` 템플릿들이 잘 맞고,  
**CMS Blog + Booking 섹션**이 같이 들어간 템플릿이면 딱 좋습니다.
