# FrameSync — 프로젝트 인수인계 (진입점)

> 새 세션/사람이 이 프로젝트를 이어받을 때 **여기부터 읽는다.**
> framer-sync(Airtable→Framer 동기화)를 구독 제품으로 상품화하는 작업.

## 30초 요약

- **무엇**: 내가 쓰려고 만든 framer-sync 를 다른 사람도 구독으로 쓰게 함.
- **왜 됨**: 어차피 내가 매일 쓰며 발전시키는 도구 → core 를 고치면 내 것도 고객 것도 같이 개선됨. 내 일이 곧 제품 개선.
- **차별점**: 공식 Framer Airtable 플러그인은 클릭 기반·수동. 이건 **일괄 upsert(엔터 한 번에 전부, 중복 없이) + Worker 자동 동기화(webhook) + 프로젝트 단위 UTM 일괄 관리.**
- **핵심 제약**: 내 프로덕션은 **절대 안 망침.** owner(나)와 customer(고객)는 코드는 공유하되 런타임은 완전 격리.
- **목표 규모**: 용돈벌이 (월 70~280만원 범위). 큰 공수 들면 지속 불가 → 운영 부담 최소가 설계 원칙.

## 지금 코드가 어디까지 됐나

`saas/tenant/` 안에 **고객 부품과 내 부품을 같은 인터페이스로** 만들어 놨다:
- 내 부품: `ownerResolver` (Doppler) — 지금 내 방식 그대로 감싼 것. 안 써도 `framer push` 무영향.
- 고객 부품: `customerResolver` (store) — 고객이 키 넣으면 store 에서 읽음. **이 부품을 고치면 모든 고객이 업데이트됨.**
- 공유 core: sync 로직(아직 platform-workers 에 있음) — 고치면 양쪽 다 개선.
