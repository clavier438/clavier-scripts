FAQ

Common questions.

Does this replace the official Framer Airtable plugin?

It runs alongside it. You keep your Framer site setup exactly as-is. FrameSync only handles the data layer — pushing records into your existing CMS collections.

Do I need a Cloudflare account?

For the always-on Worker (automatic sync on Airtable publish), yes — Cloudflare's free tier covers it. If you only want CLI-based manual syncs, no Cloudflare account needed.

What if my Airtable fields change?

FrameSync detects field changes on the next push and updates your Framer collection fields accordingly. You can also run framesync check to preview changes before they apply.

How is this different from Make or Zapier?

Make and Zapier handle general automation — they treat Framer as a black box. FrameSync speaks Framer's ManagedCollection API natively, so it understands field types, slugs, and upsert semantics that general automation tools don't.

Can I use it for multiple Airtable bases?

Yes. Each site can have its own Airtable base configuration. The CLI lets you switch between worker profiles per project.
